## Design Context

### Users
Czech-language public exploring OSA II, a civic association running autonomous cultural, sport, community and heritage-restoration projects without state funding. Three audience cuts:

- **Prospective supporters / public** — arrive curious, read a few pages, decide whether OSA is legitimate and worth caring about.
- **Existing members / stakeholders** — check organization status, documents, contact, project list.
- **Potential partners (municipal, investors for VPD1)** — scan for seriousness, scope, track record.

Use context is calm, desktop-primary, evening/weekend reading. Not urgent. Not anxious. People are deciding whether this organization is the real thing.

### Brand Personality
Three words: **independent, hands-on, literate.**

NOT: institutional, glossy, apologetic, corporate, melancholic, brutalist-dark.

Czech civic gravity: serious but unbureaucratic; print-first heritage; quiet confidence without strutting.

### Aesthetic Direction
**Swiss editorial with smart kinetism.** Tight grids, labeled blocks, hairline rules, numbered section marks, confident typography — all delivered through Atyp Special (the single font) in the existing monochrome K-scale (K0–K100). Motion is additive, restrained, and considered; every animation justifies itself as clarification of hierarchy, feedback to input, or pacing between beats. No maximalism, no brutalism, no overdrive, no dark melancholy.

**Reference fusion:**
- Interise Solutions — wordmark monumentality, clean top bar, footer-down-the-page big words.
- Osten Architects — structural labeled blocks (label column + content column + hairline between rows).
- Monolith Constructions — numbered right-column checklists with stat anchors.
- Existing `osa-design-system` ProjectCard — eyebrow, hairline, bold name, description, hairline, glyph+name+year+arrow. This is the card contract.

**Theme.** Light default. Alternating dark sections (hero, manifesto, investment hero) as established. Never a single flat tone. No scroll-linked cross-section theme inversion — each section holds its own authored theme.

**Accents.** Eight sub-project accents stay **internal**: per-card hover (already wired via `--card-accent`), sub-project subpages (section numbers, chapter marks, stat anchors, scroll-progress bar), never on parent-brand chrome. If narrowed to three secondaries: red, blue, yellow — aligns with existing `--c-red`, `--c-blue`, `--c-mustard`.

### Design Principles
1. **Structure is visible.** Labeled blocks, hairline rules, numbered sections. Never hide the grid; let it do the work.
2. **Type carries the composition.** No decorative element replaces typographic rhythm. If a heading feels weak, make it bigger or more authoritative, not more ornamented.
3. **Motion is considered, not generous.** Entrance reveals for hierarchy. Directional hover for feedback. Counter roll-up for stats. Nothing animates for its own sake.
4. **Accents whisper inside projects.** Parent brand is monochrome. Sub-project accents appear as transient, internal moments. They never become chrome.
5. **Responsive is re-laid, not squeezed.** Each breakpoint is authored. Mobile is a different document, not a smaller one.

### What to protect (hard constraints)
- Monochrome parent brand. Sharp corners (0 radius). Hairline rules (1px black, 2px for section dividers). No gradients. No glassmorphism. No border-left accent stripes >1px.
- Atyp Special as the single font family. Medium 500 body, Bold 600 headings, Italic subset.
- Editorial linter rules (no em/en dashes, no `!`, no passives, no hype). Czech diacritics correct.
- Static-first performance: no JS frameworks, no runtime CSS-in-JS, minimal JS islands, respect the weight budget.
- `prefers-reduced-motion` must disable every animation. Hover-opacity is OK; anything with transform/clip-path becomes instant.
